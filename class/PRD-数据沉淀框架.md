# PRD：课堂数据沉淀框架

> 版本：v1.0 | 日期：2026-06-16 | 作者：产品团队

---

## 一、需求背景

### 现状问题

| 问题 | 影响 |
|------|------|
| 各课堂功能的数据相互独立，随堂测、小组讨论、拍照上墙等数据不互通 | 无法形成"一个学生在一节课中的完整画像" |
| 课堂行为数据（参与次数、响应速度、主动/被动）未被采集 | 只有"对错"数据，缺乏"过程"数据 |
| 开放答题（课中选择题、拍照上墙）完全无数据留存 | 教师最常用的功能反而最缺数据 |
| 课与课之间数据不串联 | 无法回答"这个学生这学期进步了还是退步了" |
| 学校验收/续约时缺乏可量化的效果数据 | 影响项目持续性 |

### 本期定位

**只做沉淀，不做展示挖掘。**

本期目标是把"数据采集-存储-结构化"的地基打好，确保所有课堂行为数据有统一的格式和归集通道。展示层（教师课后摘要、校长看板、学生家长报告）放入下期规划。

---

## 二、需求目标

| 目标 | 衡量标准 |
|------|---------|
| 课堂行为数据全量采集 | 所有互动功能（随堂测、抢答、点名、讨论、开放答题）100%产生数据记录 |
| 数据结构统一 | 所有数据遵循统一Schema，支持跨功能聚合查询 |
| 课时级聚合可用 | 可按"学生×课时"维度查询该生该课所有行为数据 |
| 长周期串联 | 同一学生的同一知识点跨课时数据可追溯 |
| 为下期出口预留 | 数据模型可直接支撑：课后摘要、学期报告、学情诊断 |

## 三、关键决策与思考

### Q1：数据采集是否影响课堂性能？

**痛点：** 实时写入大量数据可能造成延迟，影响课堂互动体验。

**决策：**
- 课堂中数据先写入本地缓存（设备端），课后批量同步至服务端
- 关键数据（积分变动、抢答结果）实时同步；非关键数据（行为时间戳、停留时长）异步同步
- 理由：课堂体验优先，数据可以延迟但不能丢失

### Q2：开放答题（无标准答案）的数据怎么沉淀？

**痛点：** 课中选择题和拍照上墙没有录入系统，无法自动判定，但教师使用频率很高。

**决策：**
- 沉淀"行为数据"：谁参与了、何时提交、教师是否标记了（优秀/待改进）
- 不沉淀"学业数据"（因为没有标准答案）
- 课后提供可选的"补录入口"：教师可回顾本课开放题并补录正确答案，补录后数据升级为学业数据
- 理由：先保证行为层面的参与数据不丢失，学业准确度由教师异步补充

### Q3：数据存储多久？

**痛点：** 数据越多存储成本越高；但教育数据有长期参考价值。

**决策：**
- 明细数据保留当前学年（9月至次年8月）
- 聚合数据（学期报告级）永久保留
- 学年结束后明细数据归档（可恢复但不在线查询）
- 理由：平衡成本和价值，聚合数据足以支撑长期参考

### Q4：数据归属和隐私边界？

**痛点：** 学生课堂行为数据属于敏感数据，需要符合未成年人保护规范。

**决策：**
- 数据归属于学校，非平台方
- 个人级明细数据仅教师和本人可见
- 对外输出（校长/教育局）仅聚合数据，不含个人可识别信息
- 符合《个人信息保护法》《未成年人保护法》关于教育数据的要求
- 理由：合规是底线，先明确边界再采集

## 六、功能设计

### 6.1 统一数据事件Schema

所有课堂行为数据遵循统一事件格式：

```json
{
  "event_id": "uuid",
  "event_type": "quiz_answer | rush_click | roll_call | discussion_speech | open_quiz_submit | points_change",
  "timestamp": "2026-06-16T10:32:15.123Z",
  "course_session_id": "本课时唯一ID",
  "class_id": "班级ID",
  "subject_id": "学科ID",
  "student_id": "学生ID",
  "teacher_id": "教师ID",
  "payload": {
    // 各事件类型的专属数据，见下表
  },
  "context": {
    "lesson_phase": "opening | middle | closing",  // 课堂阶段（按时间自动判定）
    "interaction_mode": "active | passive | random", // 主动参与/被动参与/随机
    "device_status": "online | offline_cached"
  }
}
```

#### 各事件类型payload定义

| event_type | payload关键字段 |
|-----------|----------------|
| `quiz_answer` | question_id, knowledge_point_ids[], answer, is_correct, time_spent_ms |
| `rush_click` | session_id, click_rank, is_winner, answer(nullable), is_correct(nullable), points_earned |
| `roll_call` | strategy(random/balanced/focus), answered, teacher_rating(nullable) |
| `discussion_speech` | group_id, content_summary, ai_interaction_count, word_count |
| `open_quiz_submit` | quiz_type(choice/photo), screenshot_url(nullable), teacher_mark(nullable) |
| `points_change` | source_event_id, points_delta, reason, new_total |

### 6.2 课时级数据聚合

**触发：** 每节课结束后5分钟（教师退出课堂/下课铃声/手动结束）

**聚合输出：**

```json
{
  "session_summary": {
    "course_session_id": "xxx",
    "class_id": "xxx",
    "subject": "数学",
    "date": "2026-06-16",
    "period": 3,
    "duration_minutes": 42,
    "teacher_id": "xxx",

    "participation": {
      "total_students": 45,
      "active_students": 43,       // 至少1次互动
      "inactive_students": ["student_id_1", "student_id_2"],
      "avg_interactions_per_student": 3.2
    },

    "quiz_performance": {
      "total_quizzes": 2,
      "avg_accuracy": 0.78,
      "accuracy_vs_last_session": +0.05,  // 与上次同知识域比
      "knowledge_points_tested": ["kp_001", "kp_003"],
      "weak_points": [{"kp_id": "kp_003", "accuracy": 0.45}]
    },

    "engagement_metrics": {
      "rush_participation_rate": 0.65,
      "active_ratio": 0.4,          // 主动参与占比
      "passive_ratio": 0.6          // 被动参与占比
    },

    "is_closing_quiz": true,        // 是否有课末检测
    "closing_quiz_accuracy": 0.72
  }
}
```

### 6.3 开放答题数据采集

**现状：** 课中选择题和拍照上墙完成后无数据留存。

**变更：**

| 功能 | 新增采集内容 |
|------|-------------|
| 课中选择题 | 记录：发起时间、参与人数、每人选了什么选项、大屏截图（自动保存题面） |
| 拍照上墙 | 记录：发起时间、参与人数、每人照片URL、教师标记（优秀/待改进/未标记） |

**实现方式：**
- 系统自动在教师发起"课中选择题"时截图保存当前大屏画面（作为题面记录）
- 学生提交选项/照片时自动记录事件
- 不改变任何现有课中操作流程

---

### 6.4 教师课后补录入口

**入口：** 教师端→课后回顾→本课互动记录→"补充信息"

**页面布局：**
```
┌─────────────────────────────────────┐
│ 课后回顾 — 6月16日 第3节 数学        │
│                                       │
│ 📋 本课互动记录                       │
│                                       │
│ 1. [随堂测] 10:15 — 5题 ✅已有数据    │
│                                       │
│ 2. [课中选择题] 10:32 — 大屏截图：    │
│    ┌───────────────────┐             │
│    │ [题面缩略图]       │             │
│    └───────────────────┘             │
│    参与：42人                          │
│    选项分布：A-12 B-23 C-5 D-2        │
│    正确答案：[未设置 → 点击补录]       │
│                                       │
│ 3. [拍照上墙] 10:45 — 收到38张       │
│    已标记：优秀5张 待改进3张           │
│    [查看全部] [批量标记]              │
│                                       │
└─────────────────────────────────────┘
```

**操作流程：**
```
教师打开课后回顾 → 看到未补录的开放题 → 点击"补录正确答案" →
选择正确选项 → 系统自动计算正确率 → 数据升级为学业数据
```

**设计约束：**
- 补录为完全可选操作，不强制
- 未补录的数据仍保留行为记录（参与/未参与），仅缺学业判定
- 补录有时效：课后7天内可补录，超期不再提示

### 6.5 数据权限控制

| 数据层级 | 可见范围 | 示例 |
|---------|---------|------|
| 个人明细 | 本人+授课教师+班主任 | 张三本课所有答题记录 |
| 班级聚合 | 授课教师+教研组长+年级主任 | 3班本周数学平均正确率 |
| 年级/校级聚合 | 校长+教务处 | 初二年级智慧课堂使用率 |
| 区域聚合 | 教育局 | 本区所有学校使用覆盖率 |

## 七、异常与边界Case

| 场景 | 处理方式 |
|------|---------|
| 课中网络中断 | 数据本地缓存，网络恢复后自动同步；同步前不影响课堂功能使用 |
| 缓存数据同步失败 | 最多重试3次，仍失败则标记为"待同步"，后台告警运维处理 |
| 教师未正式"结束课堂"（直接关机） | 系统按最后一条事件时间+5分钟自动触发聚合 |
| 同一课时内教师中途切换设备 | 以course_session_id关联，不因设备切换产生新课时 |
| 学生中途更换平板（共享终端场景） | 以学生登录身份关联数据，非设备绑定 |
| 学期初无历史数据 | "进步"类指标从第2课时开始计算；首课时只记录基线 |
| 教师补录时选错答案 | 支持重新补录覆盖，以最后一次为准 |
| 数据量过大导致聚合超时 | 聚合任务设超时阈值（60秒），超时则分批聚合 |
| 学生账号合并/迁移 | 提供数据迁移工具，旧账号数据归并至新账号 |
