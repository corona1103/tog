# PRD：随堂测优化（含课末检测识别 + 课际衔接）

> 版本：v1.0 | 日期：2026-06-16 | 作者：产品团队

---

## 一、需求背景

### 现状问题

| 场景 | 当前状态 | 问题 |
|------|---------|------|
| 教师发了3次随堂测 | 每次独立统计正确率、每人数据 | 3次测试之间无关联，无法看出"学生在某知识点上是进步还是退步" |
| 学期末复盘教学效果 | 只有散落的单次测试数据 | 无法回答"这学期xx知识点全班掌握了吗" |
| 教师课末发了随堂测 | 系统视为普通随堂测 | 丢失了"出口票"的价值——无法区分"课中检测"和"课末教学效果检验" |
| 下一节课开场 | 教师凭记忆决定是否回顾 | 无数据提示"上节课哪里没讲透"，完全依赖教师个人经验 |
| 教师想看某知识点的掌握趋势 | 无法跨课时查询 | 同一知识点的多次测试分散在不同课时记录中，无法串联 |

### 产品理念参考

借鉴"出口票（Exit Ticket）"的教学理念：
- 课末检测的核心价值 = **衡量本课教学目标是否达成**
- 长周期串联 = 每节课的课末检测连成链条，形成**教学效果的趋势线**
- 课际衔接 = 上次课末的薄弱点 → 本次课开场的回顾建议

**关键约束：** 不额外增加教师操作负担。教师正常使用随堂测，系统在后台智能识别和增强。

## 二、需求目标

| 目标 | 衡量标准 |
|------|---------|
| 课末检测自动识别 | 课末15分钟内的最后一次随堂测100%被自动标记为"课末检测" |
| 知识点关联覆盖率 | ≥50%的随堂测题目有知识点标签（含AI自动推荐） |
| 课际衔接提示触达 | 教师下次上课时，100%能看到上次课末检测的薄弱点提示 |
| 教师零额外操作 | 自动识别和知识点推荐不依赖教师手动设置 |

## 三、关键决策与思考

### Q1：如何判定"课末检测"？

**痛点：** 教师不会主动打标"这是出门测"，需要系统自动识别，但课堂时间波动大，无法用固定时间点判断。

**决策：**
- 规则：一节课内的**最后一次随堂测**，且发送时间距课程结束≤15分钟 → 自动标记为"课末检测"
- "课程结束"判定：教师点击"下课"/系统检测到下一节课开始/课后无操作超过5分钟
- 边界处理：如果全课只有1次随堂测，既是过程检测也是课末检测，按课末检测权重纳入
- 理由：基于时间规则自动判定，教师无感知，系统默默增强数据价值

### Q2：知识点标签从哪来？

**痛点：** 要求教师每次出题都手动打标知识点，必然不会做。

**决策：**
- 层级1：教师从题库选题时，题库自带知识点标签（直接继承）
- 层级2：教师自编题时，AI根据题目内容自动推荐知识点标签
- 层级3：教师确认/修改AI推荐（可选操作，不确认也以AI推荐为准）
- 无标签兜底：如果以上都没有，按课件章节归类
- 理由：多层fallback确保"总有标签"，同时不依赖教师主动操作

### Q3：课际衔接提示放在哪里？

**痛点：** 提示太弱教师看不到，提示太强教师觉得烦。

**决策：**
- 下次上课时，教师小屏首页顶部显示卡片（可收起/关闭）
- 大屏不主动显示（避免学生看到"上次薄弱点"产生负面暗示）
- 教师可选择"忽略"或"投屏展示"
- 理由：仅提示教师，由教师决定是否利用这个信息

### Q4：进步/退步的判定基准是什么？

**痛点：** 不同次随堂测难度不同，直接比较正确率不合理。

**决策：**
- 相同知识点的正确率变化才纳入趋势比较
- 不同知识点的不做横向比较
- 如果同一知识点只测过1次，不显示趋势（标记"待积累"）
- 理由：只在可比场景下做比较，避免误导

### Q5：与积分系统如何联动？

**痛点：** 随堂测只有对错，缺乏激励机制。

**决策：**
- 随堂测每题答对 +1积分
- 全对额外 +2积分
- 课末检测每题答对 +2积分（权重更高，强调"最后检验"的重要性）
- 进步奖：同一知识点正确率较上次提升≥10% → +3积分
- 理由：积分联动让随堂测从"被动测试"变为"主动争分"

## 六、功能设计

### 6.1 课末检测自动识别

**判定规则：**

```
条件：
1. 本课时内发送了至少1次随堂测
2. 取最后一次随堂测
3. 该随堂测的发送时间 满足以下任一条件：
   a. 距教师点击"下课/结束课堂" ≤ 15分钟
   b. 距课程表排定的下课时间 ≤ 15分钟
   c. 是本课时的最后一次互动行为且之后无其他教学操作

→ 标记为 is_closing_quiz = true
```

**教师可见性：**
- 课后回顾页中，课末检测旁显示"📋 课末检测"标签
- 不在课中展示此标签（避免教师困惑"为什么这次随堂测多了个标记"）

---

### 6.2 知识点标签AI推荐

**触发时机：** 教师在备课/录题时保存题目后

**交互流程：**
```
教师输入/选择题目 → 保存 → 系统AI分析题目内容 →
推荐1-3个知识点标签 → 显示在题目卡片下方

┌──────────────────────────────────┐
│ Q: 解方程 x²-5x+6=0              │
│                                    │
│ 知识点标签（AI推荐）：             │
│ [一元二次方程 ✓] [因式分解法 ✓]    │
│ [+ 手动添加]                       │
└──────────────────────────────────┘
```

**规则：**
- AI推荐的标签默认生效（教师不操作也OK）
- 教师可删除不相关标签、添加其他标签
- 题库原有标签直接继承，不重复推荐
- 知识点标签来源：学科课程标准知识图谱（预置）

### 6.4 课际衔接提示卡片

**出现时机：** 教师下次给同班同学科上课时，打开教师小屏/进入课堂的前30秒

**卡片样式：**
```
┌─────────────────────────────────────┐
│ 💡 上次课末检测提示              [╳] │
│                                       │
│ 6月14日（周五）第3节 数学             │
│                                       │
│ ⚠️ 薄弱点：因式分解法（全班62%）     │
│    未达标：张三、李四、王五等8人      │
│                                       │
│ 建议：课前3分钟回顾因式分解步骤      │
│                                       │
│ [忽略]  [投屏到大屏]  [查看详情]     │
└─────────────────────────────────────┘
```

**规则：**
- 仅在教师小屏显示，大屏不主动展示
- 教师点"投屏到大屏"→ 大屏展示简化版（不含学生姓名）
- 教师点"忽略"→ 本次不再显示，不影响下次
- 卡片在课堂开始2分钟后自动收起为小图标（不持续遮挡）
- 如果上次课末检测所有知识点达标率≥85% → 不推送卡片（只在有薄弱点时才提示）

### 6.5 知识点掌握度趋势线

**入口：** 教师端→学情分析→知识点趋势（本期做基础版，下期配合数据价值挖掘深化）

**页面布局：**

```
│                                           │
│ 筛选：[一元二次方程▼]  时间：[本月▼]    │
│                                           │
│ 正确率                                    │
│ 100%│                          ●          │
│  80%│            ●    ●───────●          │
│  60%│     ●─────●                        │
│  40%│──●──                                │
│  20%│                                     │
│   0%├────┼────┼────┼────┼────┼──→       │
│     6/2  6/5  6/9  6/12 6/14 6/16       │
│                                           │

```

**规则：**
- 至少2个数据点才画趋势线
- 只有1个数据点显示"待积累"
- 支持按班级/学生个人切换查看
- 数据来源：所有含该知识点标签的随堂测题目正确率

### 6.6 积分联动规则

| 随堂测行为 | 积分 | 条件 |
|----------|------|------|
| 每题答对 | +1 | — |
| 全套答对 | +2（额外） | — |
| 课末检测每题答对 | +2 | 系统判定为课末检测的随堂测 |
| 进步奖 | +3 | 同知识点正确率较上次提升≥10% |

积分结算时机：随堂测结束（全员提交/教师手动结束）后即时结算。

## 七、异常与边界Case

| 场景 | 处理方式 |
|------|---------|
| 一节课内教师发了5次随堂测 | 最后1次标记为课末检测，前4次为过程检测 |
| 教师在下课铃后才发随堂测 | 仍按规则判定（距课程结束≤15分钟），标记为课末检测 |
| 全课只有1次随堂测 | 既标记为过程检测也标记为课末检测（双标签） |
| AI推荐的知识点标签明显错误 | 教师可删除；系统收集纠正数据用于模型优化 |
| 题目无法被AI识别知识点（如：跨学科综合题） | 标签设为"综合/未分类"，不强制归类 |
| 同一知识点2周内只测过1次 | 趋势线显示单点，不做进步/退步判定 |
| 教师连续多节课都不发随堂测 | 课际衔接卡片不推送（无数据则无提示） |
| 学科知识图谱中缺少某知识点 | 教师手动输入的标签自动进入知识图谱待审核池 |
| 转学新生无历史数据 | 从加入班级后的第1次测试开始积累，标注"新加入" |
| 课末检测正确率100%（所有知识点达标） | 课际衔接卡片不推送，但数据正常记录 |
| 教师关闭了课际衔接提示 | 提供设置项"不再提示课际衔接"（可在设置中重新开启） |
