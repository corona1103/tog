# PRD：课堂积分体系

> 版本：v1.0 | 日期：2026-06-16 | 作者：产品团队

---

## 一、需求背景

### 现状分析

**当前操作：**
- 学生参与互动（随堂测、拍照上墙、课中选择题等）后，仅有正确率统计
- 无持续性激励反馈，每次互动结束即"清零"，下一次互动重新开始
- 教师口头表扬是唯一激励方式，无法覆盖全班40+人
- 小组讨论之外，日常课堂中没有"小组"概念的持续存在

**场景问题：**

| 场景 | 问题 |
|------|------|
| 学生答对随堂测后 | 除了屏幕上显示"正确"外无任何正向反馈，答对和不答的体验差异小，缺乏成就感积累 |
| 全员投票/答题 | 学生虽然参与了操作，但因为缺乏奖励和随机性，容易变成"机械点击"，主动思考投入度不高 |
| 教师想激励课堂氛围 | 只能靠口头表扬，覆盖有限（一节课只能表扬几人）；缺少系统化的、可持续的激励工具 |
| 课堂之外回顾参与表现 | 学生无法看到自己一周内的课堂表现积累；教师无法量化追踪"谁在持续参与、谁在边缘化" |
| 综合素质评价需要过程性数据 | "课堂参与度"是评价维度之一，但当前无结构化数据支撑，只能教师凭印象填写 |
| 小组协作 | 除小组讨论功能外，日常课堂缺乏持续性的小组归属和团队激励机制 |

---

## 二、需求目标

| 目标 | 衡量标准 |
|------|---------|
| 提升学生主动参与率 | 上线后30天内，抢答/主动举手类互动参与率提升≥20% |
| 形成持续激励感 | 学生周均积分获取次数≥5次（说明每天至少有1次正反馈） |
| 促进团队协作 | 小组内成员互助行为可观测（组内落后成员进步时全组加分的触发率） |
| 为数据沉淀提供结构 | 积分流水数据100%入库，支撑下期"学生课堂表现报告" |

## 三、关键决策与思考

### Q1：积分是否对学生公开排名？

**痛点：** 公开排名可激励头部学生，但会打击后进生，导致家长投诉和学生抵触。

**决策：**
- 个人积分仅本人小屏可见，不对其他同学公开
- 小组总积分大屏公开排行（团队维度，不暴露个人）
- 理由：初中生自尊心敏感，"差生体验决定口碑"；小组维度公开可激励互助而非互踩

### Q2：积分是否与成绩/评价挂钩？

**痛点：** 挂钩则教师和学生重视；不挂钩则可能沦为摆设。

**决策：**
- 本期不与任何正式评价挂钩，定位为"课堂氛围工具"
- 积分数据沉淀后，下期作为"过程性评价"的参考维度之一（教师自主决定权重）
- 理由：先让师生无负担地用起来，验证效果后再考虑评价绑定

### Q3：积分按什么周期重置？

**痛点：** 不重置→差距越来越大→后进生绝望；频繁重置→无积累感。

**决策：**
- 默认按自然周重置（周一0点清零）
- 教师可手动配置：按单元/月重置
- 重置时生成"上周小组荣誉榜"存档（仪式感）
- 理由：一周是初中教学节奏的最小循环单位，够短不绝望、够长有积累

### Q4：小组如何分组？

**痛点：** 每节课临时分组无归属感；固定分组可能不公平。

**决策：**
- 使用"学科固定小组"（教师在后台按学科配置，一学期可调整2-3次）
- 分组建议：4-6人/组，异质分组（能力混搭）
- 可复用小组讨论功能已有的分组机制
- 理由：固定小组才能形成"我的队伍"归属感和互助责任

### Q5：积分获取规则如何防作弊/刷分？

**痛点：** 学生可能通过重复操作刷分。

**决策：**
- 每个积分来源设置单课上限（见功能设计详细规则）
- 所有积分获取由系统行为触发，学生无法主动"操作积分"
- 理由：积分是行为的副产品，不是行为的目的

## 六、功能设计

### 6.1 积分规则引擎

#### 积分获取规则表

| 行为 | 积分 | 单课上限 | 说明 |
|------|------|---------|------|
| 随堂测答对（每题） | +1 | 10分 | 基础学业行为奖励 |
| 随堂测全对（整套） | +2（额外） | 不限 | 鼓励认真完成全部 |
| 课末检测答对（每题） | +2 | 6分 | 课末检测权重更高 |
| 抢答成功+答对 | +3 | 9分（3次上限） | 奖励主动性 |
| 抢答成功+答错 | +1 | 3分 | 鼓励尝试不惩罚 |
| 被随机点名+回答 | +1 | 不限 | 仅参与即得分 |
| 幸运星被选中+回答 | +2 | 不限 | 随机公平奖励 |
| 拍照上墙被选为"优秀" | +3 | 不限 | 教师认可 |
| 小组讨论发言 | +1 | 5分 | 鼓励发言但防刷 |
| 进步奖（正确率较上次提升≥10%） | +3 | 每次随堂测判定1次 | 核心：跟自己比 |
| 教师手动加分 | +1~+5（教师选择） | 教师每课手动加分≤20次 | 口头回答等场景 |

#### 积分扣减规则

**本期不设扣分机制。** 理由：初中阶段避免惩罚性设计，只做正向激励。

#### 小组积分汇总规则

```
小组积分 = Σ(组内每位成员本周个人积分)
额外加分：组内任一成员触发"进步奖" → 全组+1（激励互帮）
```

### 6.2 学生小屏 — 个人积分动效

**触发时机：** 每次获得积分时

**交互流程：**
```
学生完成行为(答题/被点名等) → 系统判定积分 → 小屏右上角弹出动效 → 1.5秒后自动消失
```

**动效规范：**

```
┌─────────────────────────────────┐
│ [课堂内容区域]                    │
│                                   │
│                        ┌──────┐  │
│                        │ +3 ⭐ │ ← 飘字动效，1.5秒
│                        │进步奖 │    渐隐消失
│                        └──────┘  │
│                                   │
└─────────────────────────────────┘
```

| 积分类型 | 动效样式 | 颜色 |
|---------|---------|------|
| 普通得分(+1~+2) | 数字上浮渐隐 | 蓝色 |
| 高分得分(+3~+5) | 数字上浮+星星粒子 | 金色 |
| 进步奖 | 数字上浮+闪光效果+"进步奖"文字 | 橙色 |

**设计约束：**
- 动效不遮挡主要操作区域
- 不发出声音（避免课堂干扰）
- 学生不可点击积分动效（不跳转，不打断当前操作）

### 6.3 学生小屏 — 个人积分详情页

**入口：** 课堂结束后/课间，个人中心→"我的积分"

**页面布局：**
```
┌─────────────────────────────────┐
│ 我的积分                    本周  │
│                                   │
│ ┌───────────────────────────┐   │
│ │ 本周总积分：47分              │   │
│ │ 📈 较上周 +12              │   │
│ │ 🏆 我的小组：第2组 排名第1   │   │
│ └───────────────────────────┘   │
│                                   │
│ 今日获取                          │
│ ┌───────────────────────────┐   │
│ │ 10:32 随堂测答对 +1         │   │
│ │ 10:35 进步奖 +3 🌟          │   │
│ │ 10:41 抢答成功 +3           │   │
│ │ 10:50 幸运星 +2             │   │
│ └───────────────────────────┘   │
│                                   │
│ 本周闪光点                        │
│ ┌───────────────────────────┐   │
│ │ 🌟 数学连续3次进步           │   │
│ │ 🌟 英语课抢答最积极          │   │
│ └───────────────────────────┘   │
└─────────────────────────────────┘
```

**"闪光点"生成规则：**

| 闪光点类型 | 触发条件 |
|----------|---------|
| "连续N次进步" | 同一学科随堂测正确率连续3次提升 |
| "抢答达人" | 本周抢答成功≥5次 |
| "全勤参与" | 本周每节课都有互动参与记录 |
| "团队贡献者" | 本周为小组贡献积分排名组内前2 |
| "思考者" | 小组讨论中观点被AI引用/被教师选中展示 |

**设计约束：**
- 每人每周至少生成1个闪光点（系统从所有维度中找到该生最突出的1项）
- 闪光点只展示正向内容，不展示"本周最差项"

- 教师可选择"显示/隐藏"计分板（考试模式下自动隐藏）

### 6.5 教师手动加分

**入口1：** 随机点名/抢答后，大屏出现学生姓名时，旁边显示"+分"按钮

**入口2：** 老师小屏→学生列表→点击某学生→"加分"

**交互流程：**
```
教师点击"加分" → 弹出选择：+1 / +2 / +3 / +5 → 点击确认 →
学生小屏收到动效 + 大屏计分板更新
```

**操作步数：** 2步（点击加分→选分值）

---

### 6.6 小组分组配置

**入口：** 后台→班级管理→小组管理

**配置流程：**
```
选择班级 → 选择学科 → 设置小组数量 →
方式A：AI推荐异质分组（按历史学情均衡）
方式B：教师手动拖拽分组
→ 保存
```

**规则：**
- 每个学科独立分组（数学小组≠语文小组）
- 每组4-6人
- 分组后生效，直到教师再次修改
- 复用小组讨论已有的分组数据结构

---

### 6.7 积分重置与荣誉榜存档

**重置时机：** 每周一00:00自动重置

**重置流程：**
```
周日23:59 → 系统快照当前积分数据 → 生成"上周荣誉榜" →
周一00:00 → 所有小组/个人积分归零 → 新周期开始
```

**荣誉榜内容：**
- 小组排名（全部组）
- 本周进步之星（进步分最高的学生，匿名可选）
- 本周全勤奖（每节课都参与的学生数量）
- 存入历史记录，教师可随时回顾

## 七、异常与边界Case

| 场景 | 处理方式 |
|------|---------|
| 学生本课未参与任何互动 | 不得分不扣分，不做负面提示；课后摘要中对教师标记"未参与学生" |
| 学生中途加入课堂（迟到） | 从加入时刻起开始计分，不追溯之前的互动 |
| 学生转学/退出小组 | 已贡献的积分保留在小组中，该生从小组列表移除 |
| 新学生加入小组 | 从加入时刻起为新小组贡献积分 |
| 教师未配置分组 | 提示"请先配置小组分组"；个人积分功能正常，小组计分板不显示 |
| 小组人数不均（如4人组vs6人组） | 按人均积分排行（总分÷人数），避免人多组天然优势 |
| 同分排名 | 并列排名，均展示相同名次 |
| 积分重置时教师正在上课 | 不在课中重置；改为该班当天第一节课开始前重置 |
| 系统宕机/网络中断期间的行为 | 本地暂存积分变动，恢复后补发动效并同步至服务端 |
| 教师手动加分误操作 | 支持"撤销上一次加分"（5秒内可撤销） |
| 单课积分达到上限 | 达上限后该类型不再计分，但行为正常进行（不影响答题等） |
| 首次使用（无历史数据） | "进步奖"从第2次随堂测开始判定；首次参与所有行为均给基础积分 |
