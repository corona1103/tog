# PRD：课堂易用性迭代

> v2.4 | 2026-06-22 | yuj

---

## 核心原则

- **大屏 = 展示面**（学生看）；**教师小屏 = 遥控器**（教师操作）
- 所有互动功能，大屏和小屏均可发起/结束，状态实时同步
- 高频操作目标步数 ≤ 2步
- 不可逆操作保留二次确认

---

## 一、通用操作规则（全端适用）

### 1.1 预制互动的大屏快捷发起

**适用范围：** 与课件强绑定的预制互动（随堂测、单个互动题、小程序/NB实验、互动课件）

**规则：**
- 当前页码仅包含**1个**互动内容时：大屏端原有互动icon直接替换为"发起"按钮，点击即下发（省去打开互动列表容器→选择→下发的步骤）
- 当前页码包含**多个**互动内容时：保留现有交互（icon作为容器入口，点击展开互动列表，选中单个互动再下发）

**现状（需变更）：** 无论页面有几个互动，都需要点击icon→展开列表→选择→发互动，至少3步操作。

### 1.2 互动与翻页的关系

**规则：** 互动进行中，教师翻页课件 → 自动结束当前互动。

- 适用于：课中选择题、拍照上墙、小程序/NB实验
- 原因：课件页面隐含互动题目内容，翻页视为放弃当前作答
- 不适用：随堂测、单个互动题（有独立数据看板占据大屏，天然阻隔翻页动作，且教师需对题目进行深度讲解）
- 不适用：课件分享（课件分享不属于互动，不受此规则约束）

### 1.3 互动重复发起

**规则：** 所有课堂互动均支持重复发起。

- 题目类互动（随堂测、单个互动题、课中选择题）：重复发起后数据均记录，学情中心按学生维度取最优值（先取提交率高→再取正确率高）
- 操作类互动（小程序、NB实验、互动课件）：重复发起后数据均记录，不去重.ps.由于所有考勤过的学生都一定会收到且时长一致，学生是否有实际的”点按动作“是无法得知的，属于小程序内部”埋点？“，此数据实际意义不大，是否还需要记录？？cc学情pm想下

### 1.4 大屏小屏状态同步

**通用规则：**
- 大屏发起互动 → 小屏自动进入对应互动控制页面
- 小屏发起互动 → 大屏自动展示对应互动界面
- 任一端可结束互动，结束后双端同步回到课件状态
- 冲突处理：以最后一次操作为准，后到覆盖先到

**现状不满足、需改造的互动：**

| 互动类型 | 现状问题 | 目标状态 |
|---------|---------|---------|
| 课中选择题 | 大屏下发后，小屏无法结束互动；小屏无法查看结果数据/公布答案 | 任一端下发后，另一端自动同步进入互动控制态，均可公布答案/结束互动 |
| 小程序/NB实验 | 大屏下发后，小屏无法结束互动；反之亦然 | 任一端下发后，另一端可结束互动，状态双向同步 |
| 互动课件 | 大屏下发后，小屏无法结束互动；反之亦然 | 任一端下发后，另一端可结束互动，状态双向同步 |

#### 2.1.1 结果阶段 — 合并面板（变更）

作答截止后，作答分布+设置答案+公布合并为同一面板，操作路径：`看分布 → 点选正确答案 → 发布`。

**教师设置答案规则：**

| 题型 | 点击选项行为 | 发布条件 |
|------|-------------|---------|
| 单选 | 只能选中1个，再点另一个自动替换 | 选中1个 |
| 多选 | 可复选多个 | 至少选中1个 |
| 判断 | 只能选中1个（对/错） | 选中1个 |

**操作按钮（大屏+小屏一致）：**

| 按钮 | 层级 | 效果 |
|------|------|------|
| 发布答案 | 主按钮 | 至少选中1个选项 → 锁定答案 → 学生端展示对错 |
| 以上都对 | 次按钮 | 忽略选项选择，所有学生标记正确，直接发布（无二次确认） |
| 结束互动 | 弱按钮 | 直接结束，回到课件状态 |

**"以上都对"：** 本次从子菜单移至主面板按钮位，逻辑不变。适用于开放性题目或全员鼓励场景，数据标记为"全员正确（教师标记）"。误点需结束后重新发起。

#### 2.1.2 边界Case

| 场景 | 处理 |
|------|------|
| 教师未设答案直接结束互动 | 允许，本次无正确率数据 |
| 大屏与小屏同时发布答案 | 幂等操作，先到为准 |
| 多选题教师只设1个正确答案 | 允许，仅选该1个的学生正确 |

#### 2.2.1 选人上墙交互

**单人上墙：** 点击缩略图选中预览 → 点击"上墙展示"按钮投屏。已上墙学生缩略图显示"进行中"标签。

**快捷换人：** 选中另一学生后按钮变为"换人上墙"，点击即替换，无需先结束。

**多人上墙：** 预览区/缩略图均有★+按钮，点亮≥2人后主按钮自动切为"多人上墙(N)"。大屏逐个展示，左右切换。

| 操作 | 效果 |
|------|------|
| 点击缩略图 | 预览切换（不触发上墙） |
| 点击"上墙展示/换人上墙" | 投屏到大屏 |
| 点击★+ | 加入/移出上墙名单 |
| 星标≥2时点击主按钮 | 多人上墙 |
| 点击"结束" | 下墙，大屏恢复课件 |

#### 2.2.2 数据详情回看上墙

教师小屏常驻"数据详情"入口，点击进入本节课互动历史列表。

- 列表展示：互动类型、时间、参与度、正确率
- 点击条目进入二级作答详情
- **拍照上墙**条目的二级页：交互与正常上墙展示一致，支持选人、星标、上墙、结束等全部操作

#### 2.3.1 题目数据面板

**布局：** 左侧横向柱状图 + 右侧题目详情，支持全屏讲题模式。

| 区域 | 内容 | 交互 |
|------|------|------|
| 左侧柱状图 | 每行一题：题号 + 横向正确率条 + 百分比数值 | 点击切换选中题目；支持纵向滚动（兼容20+题） |
| 右侧详情 | 题干 + 作答详情（回答正确/错误/未批改/未作答，分组展示学生） | 图片作答显示缩略图，点击放大查看 |
| 全屏讲题按钮 | 右侧详情顶部 | 点击进入讲题模式 |

**柱状图颜色色阶规则：**

柱状图根据各题正确率渲染不同颜色，整体呈红→绿渐变色阶，帮助教师一眼识别薄弱题目：

| 正确率范围 | 颜色 | 色值 |
|-----------|------|------|
| 100% | 绿色 | #4caf50 |
| 85%~99% | 浅绿色 | #8bc34a |
| 60%~84% | 黄色 | #ffeb3b |
| 30%~59% | 橙黄色 | #ff9800 |
| 0%~29%（含0%） | 红色 | #f44336 |
| 尚未提交（无数据） | 灰色 | #666666 |

- 百分比文字颜色与柱状图颜色同步
- 尚未提交时柱状图统一灰色、百分比显示"--"
- 正确率为0%时仍需显示红色短条（min-width），确保可见
- 该色阶规则同时适用于全屏讲题模式的侧边栏导航和作业讲评面板

**全屏讲题模式：**

| 元素 | 说明 |
|------|------|
| 左侧题号栏 | 柱状图导航保留，点击题号快速切换 |
| 主区域 | 依次展示：题干 → 答案 → 解析 → 作答详情（白底黑字） |
| 顶部统计 | 题号后跟随：正确/错误/未批改/未作答人数 + 正确率 |
| 字号缩放 | `−` / `+` 按钮调整文本大小，适配后排投影 |
| 作答详情 | 分正确/错误两组，图片作答展示缩略图，点击弹出放大浮层 |
| 右侧工具栏 | 课件涂鸦工具（画笔/橡皮等）保持可用，不被遮挡 |
| 退出 | 右上角 ✕ 关闭，回到数据面板 |

#### 2.4.1 作答数据面板（本期新增）

**触发：** 下发互动题后，大屏自动进入数据面板。

**面板构成：**

| 区域 | 内容 |
|------|------|
| 标题栏 | "作答中" + 题型标签（选择题/解答题） + 关闭按钮（关闭=结束互动） |
| 统计区 | 已提交人数/总人数 + 提交率 + 正确率 |
| 提交进度 | 进度条 + 百分比 |
| 学生状态 | 全班姓名网格，已提交者标记区分 |
| 操作栏 | "作答详情"按钮 + "结束作答"按钮 |

**规则：**

| 规则 | 说明 |
|------|------|
| 不外显答案 | 大屏仅展示统计数据，不展示正确答案选项；操作栏固定提示"答案不在大屏显示，保护作答公平" |
| 正确率（客观题） | = 已提交学生中回答正确人数 / 已提交总人数，实时更新 |
| 正确率（主观题） | 显示"--"，待学生自批后更新 |
| 数据刷新 | 每次学生提交后立即刷新统计数值和学生状态 |

**小屏同步态：** 展示已提交/总人数/正确率 + "作答详情"按钮 + "结束作答"按钮。

| 未批改 | 人数 |
| 未作答 | 人数 |

#### 2.4.3 学生端交互

| 题型 | 流程 |
|------|------|
| 客观题 | 收到下发 → 展示题目+选项 → 选择/填写 → 提交 → 等待结果 |
| 主观题 | 收到下发 → 展示题目 → 拍照上传解答 → 提交 → 教师公布答案后对照参考答案 → 自批（"我做对了"/"我做错了"） |

#### 2.4.4 边界Case

| 场景 | 处理 |
|------|------|
| 教师未点"作答详情"直接结束互动 | 允许，数据面板中已收集的提交数据正常记录 |
| 学生提交后教师翻页 | 不自动结束互动（同规则1.2），大屏仍停留在数据面板 |

| 主观题全部学生均未自批 | 正确率持续显示"--" |
| 重复下发同一题目 | 允许，同规则1.3，学情取最优值 |
| 下发时无学生在线 | 正常下发，面板显示 0/0，学生上线后自动收到题目 |
| 作答过程中学生掉线 | 已提交的答案不受影响；未提交的学生重连后可继续作答 |
| 大屏小屏同时点击"结束作答" | 幂等操作，先到为准 |

| 4 | 右侧详情顶部保留"查看原卷"+"全屏讲题"入口 | P0 |
| 5 | 全屏讲题展示：题干+答案+解析+字号缩放+左侧题号导航+正确率色阶，样式同2.3.1 | P0 |

### 2.9 随机点名动画时长优化（变更）

**客户反馈：** 随机点名功能教师使用频率高、认可度高，但点名动画时间过长（约8秒），课堂节奏拖沓。

| # | 变更内容 | 优先级 |
|---|---------|--------|
| 1 | 点名滚动动画总时长由~8秒缩短至~3秒 | P0 |

### 2.10 排行榜单（新增）

**定位：** 课堂工具箱内的独立功能，教师在任意时刻（通常为课末）手动触发，大屏展示本节课个人与小组的爱学值排行。

> 爱学值 = 产品内部"激励经验值"的学生端外显名称。积分体系详细设计见《PRD-课堂积分体系》。

#### 2.10.1 功能入口

| 入口位置 | 操作 |
|---------|------|
| 大屏工具箱 | 点击"🏆 排行榜单"按钮 |
| 教师小屏工具箱 | 点击"🏆 排行榜单"按钮 |
| 顶部场景Tab | 点击"排行榜单"Tab |

#### 2.10.2 大屏展示

**布局：** 全屏深色渐变覆盖层，顶部Tab切换，中部排行列表。

**不展示的内容：** 课堂标题、积分规则说明（本期仅聚焦排行本身）。

**个人排行Tab：**

| 元素 | 说明 |
|------|------|
| 展示范围 | 班级前10名 |
| 每行信息 | 排名（前3名用奖牌🥇🥈🥉） + 学生姓名 + 正确率条形图 + 本节课爱学值 |
| 底部提示 | "仅展示班级前10名 · 每位同学可在个人端查看自己的排名" |

**小组排行Tab：**

| 元素 | 说明 |
|------|------|
| 展示条件 | 仅当班级已配置小组分组数据时展示此Tab；未配置时不显示Tab |
| 每行信息 | 排名（第1名用🏆） + 小组名 + 本节课小组爱学值 + 正确率条形图 |
| 小组成员 | 每组下方展示全部组员姓名（顿号分隔） |

**数值说明：**
- 个人爱学值 = 本节课该学生通过各互动行为获得的激励经验值总和
- 小组爱学值 = 本节课组内全部成员个人爱学值之和

#### 2.10.3 涉及增加爱学值的功能清单

以下课堂互动行为会为学生增加爱学值（详细分值见《PRD-课堂积分体系》6.1 积分规则引擎）：

| 行为来源 | 具体行为 | 分值 |
|---------|---------|------|
| 随堂测 | 每题答对 | +1 |
| 随堂测 | 整套全对（额外） | +2 |
| 课末检测 | 每题答对 | +2 |
| 抢答 | 抢答成功+答对 | +3 |
| 抢答 | 抢答成功+答错 | +1 |
| 随机点名 | 被点名+回答 | +1 |
| 幸运星 | 被选中+回答 | +2 |
| 拍照上墙 | 被选为"优秀" | +3 |
| 小组讨论 | 发言 | +1 |
| 进步奖 | 正确率较上次提升≥10% | +3 |
| 教师手动 | 教师点击加分 | +1~+5 |

#### 2.10.4 教师小屏控制面板

| 元素 | 说明 |
|------|------|
| 标题 | "🏆 排行榜单" |
| Tab切换 | 个人排行 / 小组排行（受分组数据控制） |
| 统计摘要 | 本节课互动数据（随堂测次数、互动题次数等）、全班平均爱学值、最高爱学值+姓名 |
| 关闭按钮 | "关闭榜单"，点击后回到课件状态 |

#### 2.10.5 学生端展示

教师触发排行榜单后，学生小屏同步展示个人本节课成绩：

| 元素 | 说明 |
|------|------|
| 本节爱学值 | 大字突出展示，如"+24" |
| 获取明细 | 小字说明各行为贡献，如"随堂测满分+5 · 互动作答+2×3 · …" |
| 我的班级排名 | "第N名" |
| 我的小组排名 | "第N名"（有分组时展示） |

#### 2.10.6 与现有能力的关系

| 关联能力 | 说明 |
|---------|------|
| 个人中心 — 爱学值明细 | 学生下课后可在个人中心查看爱学值明细记录表，**本期直接复用已有能力**，将课堂内各行为产生的分值变化写入明细流水即可 |
| 课堂总结报告 | 本期**不新增**独立的课堂总结报告能力，排行榜单仅作为课堂内实时展示 |
| 小组计分板 | 排行榜单与课堂右侧常驻小组计分板数据一致，排行榜单为全屏展示态 |

#### 2.10.7 边界Case

| 场景 | 处理 |
|------|------|
| 班级未配置小组分组 | 不展示"小组排行"Tab，仅展示个人排行 |
| 本节课无人获得爱学值 | 个人排行展示全部0分，保留排行结构 |
| 并列分数 | 并列排名，均展示相同名次 |
| 教师多次打开排行榜单 | 每次展示实时最新数据 |
| 课堂中途学生加入 | 从加入时刻起计分，排行实时更新 |

## 三、异常与边界Case

| 场景 | 处理方式 |
|------|---------|
| 教师小屏与大屏断联 | 小屏提示"连接中断"，操作暂存，恢复后自动同步 |
| 大屏小屏同时操作 | 以最后一次操作为准，后到覆盖先到 |
| 学生设备离线误判 | 心跳30秒超时标记离线，状态概览实时更新 |
| 多教师协同教学 | 课程表绑定教师为主控，其他教师为观察者（只看不操作） |
| 教师小屏关机/没电 | 不影响大屏运行，教师可回大屏直接操作 |
