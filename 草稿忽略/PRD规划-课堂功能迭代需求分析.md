# 课堂功能迭代 — PRD规划需求分析

> 基于现有产品能力 × 第一性原理分析 × 初中学段特性

---

## 分析原则

1. **映射假设分析**：所有需求必须与《课堂环节第一性原理分析》中的洞察一致，不背离已验证的假设
2. **师生特色优先**：教师易用性（操作步数最少化）、学生内驱力（非外部压力驱动）
3. **跨学段兼容**：标注哪些功能需要按学段做策略/UI定制
4. **政策合规**：普惠公平、屏幕时间控制、不加重师生负担

---

## 一、应新增的稀缺功能

### 【新增1】出门测（Exit Ticket）

| 维度 | 说明 |
|------|------|
| **对应洞察** | 第一性原理：屏幕是采集器；P2优先级功能；"数据能不能在期末汇报中用"决定复购 |
| **为什么稀缺** | 现有"随堂测"是通用测验工具，缺乏课末定向检测+教学目标关联+课际衔接的产品化封装 |
| **核心定义** | 课堂最后3-5分钟，2-3题，紧扣本课教学目标，全班必答，即时反馈达成率 |
| **师生价值** | 教师：下课30秒内知道"今天讲的学生掌握了多少"；学生：无排名压力，跟自己的目标比 |
| **与现有能力关系** | 复用随堂测底层（题目下发/回收/统计），新增：教学目标绑定、课际趋势线、达成率看板 |

**产品形态要点：**
- 备课时：1键关联本课教学目标 → AI推荐对应检测题（每个目标1题）
- 课中：工具箱1键触发（或课件末页自动提示）
- 课后：自动生成"本课达成率"卡片，标记薄弱目标，建议下节课回顾内容
- 学期维度：每课出门测串联 → 教学目标达成趋势曲线 → 可纳入学期报告

**跨学段定制：**
- 初中：2-3题，纯选择/判断，3分钟内完成，不计入成绩
- 高中：可扩展为3-5题，支持简答，可选计入过程性评价

---

### 【新增2】课堂积分 / 小组竞赛计分板

| 维度 | 说明 |
|------|------|
| **对应洞察** | P2"小组竞赛计分板"；假设#1修正——参与度的本质是思维卷入，游戏化应服务于思考而非点击 |
| **为什么稀缺** | 当前缺乏持续性的课堂激励机制，随机点名是瞬时的，小组讨论是单次的，没有跨课时的积累感 |
| **核心定义** | 大屏常驻的小组/个人积分板，教师可1键加分（口头回答、协作表现等），学生看到积累 |
| **内驱力设计** | 不排名个人分数（避免后进生打击），只展示小组总分 + 个人"进步徽章"（跟自己比） |

**产品形态要点：**
- 大屏：课堂右侧常驻小组积分条（可折叠），教师点名/互动后1键加分
- 加分触发：随机点名回答正确、拍照上墙被选为优秀、出门测进步等
- 重置周期：按周/单元重置，避免差距越拉越大
- 不消耗学生屏幕时间（仅大屏展示）

**跨学段定制：**
- 初中：小组积分为主，弱化个人竞争，强调"团队荣誉感"
- 高中：可开放个人积分排行榜（学生自主选择是否公开）

**政策合规：**
- 不暴露具体分数排名，只展示小组维度
- 符合"不加重学生心理负担"要求
- "进步量"对比（跟自己的历史比）而非绝对值排名

---

### 【新增3】课后5分钟摘要（教师专属）

| 维度 | 说明 |
|------|------|
| **对应洞察** | 假设#4："实时数据对教师有用"部分错误——有用的是"课后5分钟摘要"而非实时仪表盘 |
| **为什么稀缺** | 现有随堂测有即时统计，但缺乏课后自动汇总的"1张卡片看清本课全貌"的产品 |
| **核心定义** | 下课后自动生成，推送到教师端（小屏/微信/钉钉），1张卡片，30秒读完 |

**卡片内容：**
```
┌────────────────────────────────┐
│ 📋 第3节课 · 数学 · 课后摘要    │
│                                  │
│ 参与概况：43/45人参与互动         │
│ 随堂测正确率：78%（较上次+5%）    │
│ 出门测达标率：目标1✅ 目标2⚠️68%  │
│                                  │
│ ⚠️ 关注学生：张三、李四（连续2次  │
│    目标2未达标）                  │
│                                  │
│ 💡 建议：下节课前3分钟回顾xx知识点│
│                                  │
│ [查看详情]  [推送针对性练习]      │
└────────────────────────────────┘
```

**设计原则：**
- 自动生成，教师零操作
- 不需要"实时盯着看"，而是课间/午休时glance一眼
- 关注"变化量"（进步/退步）而非绝对值

**跨学段定制：**
- 初中：摘要推送到教师端即可，不推送给家长
- 高中：可选推送精简版给学生本人（自我管理）

---

### 【新增4】学期数据报告自动生成

| 维度 | 说明 |
|------|------|
| **对应洞察** | "数据能不能在期末汇报中用"决定复购；校长向教育局汇报需要数据 |
| **为什么稀缺** | 现有数据停留在"每次互动的即时统计"，缺乏学期/学年维度的自动汇总 |
| **核心定义** | 学期末自动生成多维报告，供教师、教研组长、校长、教育局不同层级使用 |

**报告层级：**

| 使用者 | 报告内容 |
|--------|---------|
| 教师 | 本班学情趋势、教学目标达成率变化、高频错题TOP10 |
| 教研组长 | 跨班对比（脱敏）、学科薄弱点分布、平台功能使用率 |
| 校长/管理层 | 全校使用覆盖率、教师信息化应用率、教学效果相关性数据 |
| 教育局 | 区域数据汇总、项目成效证明、政策合规指标达标情况 |

**政策合规：**
- 班级对比数据脱敏，不用于教师考核排名
- 强调"使用率"和"趋势"而非"绝对成绩"
- 符合教育公平——展示缩小差距的数据，而非拉大差距的排名

---

### 【新增5】学科模板（差异化课型支持）

| 维度 | 说明 |
|------|------|
| **对应洞察** | 假设#3确认"每个学科需要相同的平板集成度"是错误的 |
| **为什么稀缺** | 当前所有学科使用相同的工具箱和交互方式，未考虑学科场景差异 |
| **核心定义** | 按学科/课型预设"推荐使用模板"，告诉教师"这节课建议用哪些工具、在什么时间点用" |

**示例模板：**

| 学科 | 推荐模板 | 屏幕时间分配 |
|------|---------|-------------|
| 数学（练习课） | 导入投票(2min) → 纸上练习 → 拍照上传(2min) → 典型错误投屏(3min) → 出门测(3min) | ~10min |
| 语文（阅读课） | 小组讨论(8min) → 观点投屏(2min) → 出门测(2min) | ~12min |
| 英语（听说课） | AI对话练习(5min) → 课中选择题(2min) → 拍照上墙-口语评分(3min) | ~10min |
| 理科（实验课） | 虚拟实验(5min) → 拍照记录实验现象(2min) → 出门测(3min) | ~10min |

**产品形态：**
- 备课时：选择课型 → 系统推荐工具组合和时间分配 → 教师可调整
- 授课时：工具箱按模板排序，高亮"当前阶段推荐工具"
- 非强制：模板只是建议，教师随时可自由使用任何工具

**跨学段定制：**
- 初中：模板更固化，减少选择负担（"照着用就对了"）
- 高中：模板更灵活，仅作参考，教师自主度高

## 二、现有功能优化方向

### 【优化1】随堂测 → 增加"教学目标关联"

| 现状 | 问题 | 优化方向 |
|------|------|---------|
| 题目录入系统后下发，统计正确率 | 数据是"离散的"——每次测验独立，无法串联分析 | 题目打标：关联知识点/教学目标 → 支持跨课时的学情追踪 |

**具体改进：**
- 录题时增加"关联知识点"字段（可选，AI辅助推荐）
- 数据看板从"单次正确率"升级为"知识点掌握度趋势"
- 为出门测和学期报告提供数据基础

**易用性保障：**
- 关联知识点为可选项，不关联也能正常使用（不增加门槛）
- AI自动推荐知识点标签，教师1键确认即可

---

### 【优化2】开放答题（课中选择题/拍照上墙）→ 补数据断层

| 现状 | 问题 | 优化方向 |
|------|------|---------|
| 题目未录入系统，无法自动判对错 | 教师用得多（因为简单），但数据完全丢失，不进入学情分析 | 课后补录+AI辅助，弥合数据断层 |

**具体改进：**
- 课中选择题：大屏点击后，系统自动截图保存题面 → 课后教师1键标注正确答案 → 数据补录进学情
- 拍照上墙：教师课中点"优秀"/"待改进"标记 → 课后AI辅助识别共性错误
- 补录为可选操作，不补录不影响功能使用

**映射洞察：**
- 尊重假设#8"功能越多竞争力越强在初中市场错误"——不增加课中操作复杂度，优化放在课后
- 符合"教师愿不愿意用才是生死线"——课中依然极简，数据增值放在课后异步环节

---

### 【优化3】小组讨论 → 屏幕时间优化 + 安静学生参与

| 现状 | 问题 | 优化方向 |
|------|------|---------|
| 语音+AI对话，功能丰富 | 1. 屏幕时间消耗大（可能5-10min）；2. 依赖语音表达，性格内向学生参与度低；3. 麦克风常开的隐私顾虑 | 增加文字观点输入 + 讨论时间建议 + 隐私模式 |

**具体改进：**
- 增加"文字输入观点"入口（与语音并行，不强制二选一）
- 大屏讨论面板增加"建议剩余时间"提示（帮助教师控制节奏）
- 麦克风：进入讨论后明确提示"麦克风已开启"，并提供手动关闭选项
- 讨论贡献统计：除"发言数"外，增加"观点被引用数"、"AI互动深度"等多维指标（不只靠说话多少衡量参与）

**映射洞察：**
- 假设#7"互动频次越高课堂越好"是错误的——讨论质量>讨论数量
- 假设#1修正——思维卷入不等于嘴巴多动，文字深思也是高质量参与
- "差生体验决定口碑"——内向≠不参与，需要多元参与通道

**跨学段定制：**
- 初中：默认显示任务引导卡（结构化讨论），AI引导更主动
- 高中：可选自由讨论模式，AI仅在被@时响应

**政策合规：**
- 实时收音需明确告知学生，符合未成年人隐私保护
- 讨论记录存储需符合数据安全要求，可配置自动清理周期

---

### 【优化4】工具箱 → 场景感知 + 减少选择负担

| 现状 | 问题 | 优化方向 |
|------|------|---------|
| 工具箱平铺展示所有工具 | 教师需要自己判断"现在该用什么"，认知负担随功能增多而增加 | 场景感知推荐 + 快捷入口 |

**具体改进：**
- 如果使用了"学科模板"，工具箱顶部显示"当前推荐"（如：课末阶段推荐"出门测"）
- 高频工具（教师最常用的3个）置顶显示
- 支持"长按大屏空白处"快速调出最近使用的工具（类似快捷键）

**映射洞察：**
- "功能的最少操作步数比功能丰富度重要10倍"
- 1步操作>80%使用率，5步以上<10%使用率

---

### 【优化5】随机点名 → 增加教学策略性

| 现状 | 问题 | 优化方向 |
|------|------|---------|
| 纯随机点名 | 等概率随机，缺乏教学意图——教师有时想点"这道题错的学生"或"最近不活跃的学生" | 策略点名模式 |

**具体改进：**
- 保留"纯随机"作为默认模式（简单，覆盖"制造全员警觉"的需求）
- 新增可选策略：
  - "关注模式"：优先点最近3次互动未参与的学生
  - "巩固模式"：优先点上次出门测未达标的学生
  - "均衡模式"：确保一周内每人至少被点到1次
- 策略对学生不可见（不暴露"你被重点关注了"）

**跨学段定制：**
- 初中：策略模式需保护学生自尊心——被点名不等于"差"，需配合鼓励性话术模板
- 高中：可开放更多策略选项给教师

---

### 【优化6】AI人物对话 / AI智能体 → 合规边界明确化

| 现状 | 问题 | 优化方向 |
|------|------|---------|
| AI对话能力在讨论和独立对话中可用 | 1. 缺乏使用时间限制；2. AI回复的教育适切性保障；3. 与"30%屏幕时间"的关系不明确 | 限时窗口 + 教师可控 + 内容围栏 |

**具体改进：**
- AI对话限定为"教师发起的时间窗口内可用"，非学生随时可触发
- AI回复策略：苏格拉底式追问优先（已有），增加"本课知识边界"约束（不超纲回答）
- 教师可预设AI的"人设+边界"（如：这节课只讨论《故都的秋》，不回答其他话题）
- 使用时长自动纳入屏幕时间统计

**政策合规：**
- AI不替代教师教学（定位为"辅助"）
- 不向学生提供超纲内容
- 符合《生成式人工智能服务管理暂行办法》对未成年人的保护要求

## 三、不建议做的功能（负面清单）

基于假设分析和初中学段特性，以下功能**不纳入本期规划**：

| 功能 | 不做的理由 | 对应洞察 |
|------|----------|---------|
| AR/VR互动 | 新鲜感消退快、数据零沉淀、成本高、是展示项目非常态化工具 | P5最低性价比 |
| 实时学情仪表盘（教师授课时看） | 教师无法授课同时解读数据，反而分心 | 假设#4 |
| 个人分数排名/公开排行榜 | 后进生持续暴露失败→抵触→家长投诉→弃用 | "差生体验决定口碑" |
| 复杂分层推题（课中） | 需教师预分组+题库配置，5步以上操作，初中教师接受度极低 | 假设#8 + 操作步数理论 |
| 学生自由浏览内容 | 屏幕是采集器不是播放器；自由浏览=注意力分散=课堂管控风险 | 核心设计原则 |
| 家长端实时课堂数据 | 初中不推送实时数据给家长（增加焦虑），只做周/月摘要 | 政策合规+减负 |

## 六、核心设计原则（贯穿所有需求）

| # | 原则 | 来源 |
|---|------|------|
| 1 | **屏幕是采集器，不是播放器** | 第一性原理核心洞察 |
| 2 | **1步操作 > 3步操作 > 5步操作** | 教师使用概率模型 |
| 3 | **课后异步 > 课中实时**（对教师而言） | 假设#4修正 |
| 4 | **进步量 > 绝对值**（对学生而言） | "差生体验决定口碑" |
