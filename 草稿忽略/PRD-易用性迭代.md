# PRD：易用性迭代（多端课堂操控体验优化）

> 版本：v1.0 | 日期：2026-06-16 | 作者：产品团队

---

## 一、需求背景

### 现状问题

| 场景 | 当前操作 | 问题 |
|------|---------|------|
| 教师走动授课时想发起互动 | 必须走回讲台操作大屏 | 打断授课节奏，物理距离限制了互动频率 |
| 教师想知道"学生现在屏幕显示什么" | 无法从自己小屏确认学生端状态 | 信息不对称，无法确认指令是否下达成功 |
| 发起随堂测 | 工具箱→找到功能→选题/配置→发放 | 步骤过多（3-5步），降低使用意愿 |
| 讨论中需要临时回到课件讲解 | 必须先"结束讨论"→回课件→重新发起讨论 | 不可逆操作，讨论数据丢失 |
| 课件播放中想使用工具箱功能 | 点开工具箱→找到功能→使用→关闭工具箱→回课件 | 工具箱遮挡课件，操作跳出感强 |
| 多个功能同时运行 | 例如积分计分板+课件播放+倒计时 | 各功能间缺乏空间协调，相互遮挡 |

### 核心原则

> "功能的最少操作步数比功能丰富度重要10倍"
>
> 1步操作=80%使用率 / 3步=50% / 5步以上<10%

本期目标：将高频操作的步数压缩到1-2步，将教师小屏打造为"课堂遥控器"。

---

## 二、需求目标

| 目标 | 衡量标准 |
|------|---------|
| 高频功能操作步数≤2步 | 随堂测发放、抢答发起、点名、积分加分均≤2步到达 |
| 教师小屏可覆盖所有课堂操作 | 教师无需走回讲台即可完成100%课堂互动操作 |
| 学生端状态对教师可见 | 教师小屏任何时刻可看到"学生当前处于什么状态" |
| 功能切换无损 | 讨论/课件/工具可暂挂切换，不丢失进度 |

---

## 三、关键决策与思考

### Q1：教师小屏定位是什么？

**痛点：** 当前教师小屏和大屏功能有重叠但不完全一致，教师不确定"该在哪个端操作"。

**决策：**
- 大屏 = 面向全班的"展示面"（学生看的）
- 教师小屏 = 面向教师个人的"遥控器"（教师自己看和操作的）
- 所有大屏展示内容，教师小屏均可触发/控制
- 教师小屏有独享信息：学生状态概览、下一步提示、操作日志
- 理由：明确分工避免困惑，让教师建立"小屏在手=掌控全局"的心智

### Q2：快捷操作是否会误触？

**痛点：** 操作步数越少，误触风险越大。

**决策：**
- 1键触发类操作（抢答/点名）：允许3秒内取消
- 不可逆操作（结束讨论/重置积分）：保留二次确认
- 理由：高频操作追求速度（误触可撤销），低频+不可逆操作保留安全门槛

### Q3：工具箱是否需要重新设计信息架构？

**痛点：** 工具箱功能越加越多，扁平列表不适合快速寻找。

**决策：**
- 保留当前分类架构（互动工具/资源管理/课堂管理）
- 新增"最近使用"置顶区（显示最近3次使用的工具）
- 新增教师小屏底部"快捷栏"（可自定义4个高频工具）
- 理由：不大改架构（避免老用户不适应），通过"快捷通道"解决效率问题

## 六、功能设计

### 6.1 教师小屏 — 快捷操作栏

**位置：** 教师小屏底部常驻，课堂模式下始终可见

**布局：**
```
┌─────────────────────────────────────┐
│                                       │
│          [课堂内容区域]                │
│                                       │
│                                       │
├─────────────────────────────────────┤
│  [随堂测]  [抢答]  [点名]  [更多▼]   │ ← 快捷栏
└─────────────────────────────────────┘
```

**规则：**
- 默认显示3个快捷按钮 + 1个"更多"入口
- 教师可自定义快捷栏内容（从所有工具中选择3个）
- 点击快捷栏按钮 = 直接执行（沿用上次配置），长按 = 进入配置

**操作步数对比：**

| 操作 | 优化前 | 优化后 |
|------|--------|--------|
| 发起抢答 | 工具箱→AI功能→抢答→配置→发起（5步） | 快捷栏点"抢答"→发起（1-2步） |
| 随机点名 | 工具箱→课堂管理→随机点名（3步） | 快捷栏点"点名"（1步） |
| 发起倒计时 | 工具箱→课堂管理→倒计时→设置时间→开始（4步） | 快捷栏点"更多"→倒计时→开始（2步） |


### 6.3 教师小屏 — 遥控大屏操作

**原则：** 教师小屏可触发的操作 ≥ 大屏可触发的操作

**新增遥控能力：**

| 操作 | 当前 | 优化后 |
|------|------|--------|
| 翻页课件 | 仅大屏可翻 | 小屏左右滑动=翻页 |
| 打开/关闭计分板 | 仅大屏点击 | 小屏快捷栏"更多"→计分板开关 |
| 展示/隐藏答案 | 仅大屏操作 | 小屏随堂测结果页→"大屏展示答案" |
| 全屏展示某学生作品 | 仅大屏选择 | 小屏浏览学生作品→"投屏到大屏" |
| 切换考试模式 | 仅大屏 | 小屏"更多"→"考试模式" |

### 6.4 工具箱 — "最近使用"置顶

**变更：** 工具箱打开后，顶部增加"最近使用"区域

```
┌────────────────────────┐
│ 工具箱              ╳  │
│                         │
│ 最近使用                │
│ [随堂测] [抢答] [点名]  │
│ ─────────────────────  │
│ 互动工具                │
│ [拍照上墙] [课中选择题] │
│                         │
│ 资源管理                │
│ [课程资料] [云盘资料]...│
│                         │
│ 课堂管理                │
│ [发起考勤] [解锁]...   │
└────────────────────────┘
```

**规则：**
- 展示最近3次使用的不同工具（去重）
- 首次使用（无历史）时不显示此区域
- "最近使用"统计维度为该教师个人（非班级/学科）

### 6.5 功能暂挂/恢复机制

**问题：** 小组讨论中教师想临时回到课件讲解2分钟，当前必须结束讨论。

**解决方案：**

```
讨论进行中 → 教师点"暂挂讨论" → 大屏回到课件 / 学生小屏显示"讨论暂停中"
→ 教师讲解完毕 → 点"恢复讨论" → 回到讨论（数据+状态完全保留）
```

**支持暂挂的功能：**

| 功能 | 暂挂时学生小屏 | 暂挂时大屏 |
|------|--------------|----------|
| 小组讨论 | 显示"讨论暂停，请注意老师讲解" | 回到课件 |
| 随堂测（进行中） | 计时暂停，保留已作答内容 | 回到课件 |

**约束：**
- 同时只能暂挂1个功能
- 暂挂超过10分钟自动提示教师"是否结束该功能"
- 暂挂状态下，教师小屏顶部显示"⏸️ 讨论已暂挂 [恢复]"

---

### 6.6 随堂测快速发放

**场景：** 教师临时想发一个随堂测，但不想经过"选题→配置→发放"的完整流程。

**优化方案：**

**方式A：预设题组1键发放**
```
备课时：教师已将题目嵌入课件或创建了随堂测
授课时：快捷栏点"随堂测" → 显示"待发放题组列表" → 点击任意一组 → 立即发放
```
步数：2步（点击快捷栏→选择题组）

**方式B：课件内嵌互动自动提示**
```
教师翻到包含互动题的课件页 → 教师小屏底部自动弹出：
┌─────────────────────────────────────┐
│ 💡 本页有互动题（3题）  [发送到学生] │
└─────────────────────────────────────┘
```
步数：1步（点击"发送到学生"）

---

### 6.7 操作即时反馈

**问题：** 教师点了"发送"后不确定学生是否收到了。

**优化：** 每个操作完成后，教师小屏底部显示Toast确认：

| 操作 | Toast内容 | 展示时长 |
|------|----------|---------|
| 发送随堂测 | "✅ 已发送至45人" | 2秒 |
| 发起抢答 | "✅ 抢答已开始" | 2秒 |
| 随机点名 | "✅ 已选中：张小明" | 2秒 |
| 锁定小屏 | "✅ 已锁定45台设备" | 2秒 |
| 解锁小屏 | "✅ 已解锁" | 2秒 |

---

### 6.8 误操作撤销机制

**适用范围：** 1键触发类操作（非不可逆操作）

**交互方式：**
```
教师执行操作 → 底部弹出Snackbar：
┌─────────────────────────────────────┐
│ 已发起抢答              [撤销](3秒) │
└─────────────────────────────────────┘
→ 3秒内点"撤销" → 操作回退，学生端无感知
→ 超过3秒 → Snackbar消失，操作生效
```

**可撤销的操作：**
- 发起抢答
- 发起随机点名
- 发送随堂测
- 教师手动加分

**不可撤销的操作（保留二次确认弹窗）：**
- 结束小组讨论
- 重置小组积分
- 结束课堂

## 七、异常与边界Case

| 场景 | 处理方式 |
|------|---------|
| 教师小屏与大屏网络断联 | 小屏显示"连接中断"提示，操作暂存，恢复后自动同步 |
| 教师同时操作大屏和小屏 | 以最后一次操作为准（后到的覆盖先到的），不产生冲突 |
| 暂挂功能后教师忘记恢复 | 10分钟后弹出提示"小组讨论已暂挂10分钟，是否结束？" |
| 快捷栏自定义后重置 | 仅在教师主动修改时改变，不随系统更新重置 |
| 学生设备离线但教师小屏显示在线 | 设备心跳30秒超时则标记离线，状态概览实时更新 |
| 教师小屏电量低/关机 | 不影响大屏功能运行；教师可回到大屏直接操作 |
| 撤销操作时学生已看到题目 | 撤销后学生端页面自动关闭，显示"教师已取消本次操作" |
| 多位教师同时在一个班级授课（协同教学） | 以课程表绑定教师为主控，其他教师为观察者（只能查看不能操作） |
