# PRD：随机点名策略优化

> 版本：v1.0 | 日期：2026-06-16 | 作者：产品团队

---

## 一、需求背景

### 现状问题

| 场景 | 当前状态 | 问题 |
|------|---------|------|
| 教师使用随机点名 | 纯等概率随机 | 一节课可能同一学生被点到2次，而部分学生整周未被点到 |
| 教师想关注某些学生 | 只能记在脑子里，刻意点名 | 对学生来说能感知到"被针对"，产生抵触 |
| 课堂参与度不均 | 活跃学生主动参与多，沉默学生被忽略 | 纯随机不能补偿参与度差异 |
| 与抢答功能配合 | 抢答奖励主动性 | 缺乏对"非主动型学生"的覆盖机制，两极分化可能加大 |

### 设计目标

将"随机点名"从纯随机工具升级为"有教学策略的覆盖工具"，同时引入"幸运星"变体，让被选中成为正向体验而非压力事件。

---

## 二、需求目标

| 目标 | 衡量标准 |
|------|---------|
| 参与覆盖均衡性 | 使用"均衡模式"时，一周内每位学生至少被互动触达1次 |
| 教师操作无增加 | 策略切换≤1步，默认模式教师无需任何额外操作 |
| 学生体验正向 | 被选中的学生获得积分奖励，框架为"机会"而非"审问" |
| 与抢答互补 | 抢答覆盖主动型学生，幸运星覆盖非主动型学生，两者合计覆盖全班 |

---

## 三、关键决策与思考

### Q1：默认策略是哪个？

**痛点：** 如果默认纯随机，教师可能不知道有其他策略；如果默认均衡模式，教师可能觉得"不够随机"。

**决策：**
- 默认"均衡模式"
- 理由：均衡模式在体验上和纯随机几乎一致（学生看不出区别），但后台保证了覆盖公平性。教师不需要知道细节差异即可受益。

### Q2：策略是否对学生透明？

**痛点：** 如果学生知道"快轮到我了"，会提前紧张；如果知道"刚被点过不会再点"，会放松警惕。

**决策：**
- 策略完全对学生不可见
- 大屏动效与纯随机完全一致（滚动姓名→停留→选中）
- 学生无法推断自己被选中的概率
- 理由：保持"随时可能被叫到"的适度紧张感是点名的核心教学价值

### Q3："幸运星"与"随机点名"是同一功能还是两个功能？

**痛点：** 两个功能入口太多，教师混淆；合并又怕丢失各自特色。

**决策：**
- 统一为1个入口："点名/幸运星"
- 入口内提供两种"皮肤"：
  - "点名"皮肤：大屏显示为常规点名动效（较严肃）
  - "幸运星"皮肤：大屏显示为幸运星降临动效（较活泼）
- 后台逻辑完全相同，只是前端动效和氛围不同
- 教师根据课堂氛围自行选择（严肃提问用"点名"，活跃氛围用"幸运星"）
- 理由：减少功能数量，一个入口两种表达

## 六、功能设计

### 6.1 点名策略引擎

#### 三种策略定义

| 策略 | 选中逻辑 | 适用场景 |
|------|---------|---------|
| **均衡模式**（默认） | 优先选本课/本周互动参与最少的学生；已被点名的学生概率降低但不归零 | 日常使用，保证全员覆盖 |
| **关注模式** | 优先选"连续N课参与度低"或"近期随堂测正确率下降"的学生 | 教师有意识想关注后进/退步学生 |
| **纯随机** | 等概率随机，不考虑历史 | 教师明确想要纯随机效果时 |

#### 均衡模式算法（简化描述）

```
输入：全班学生列表 + 每人本课参与次数 + 本周参与次数
权重计算：
  - 本课参与次数为0 → 权重×3
  - 本周参与次数低于平均值 → 权重×2
  - 上一次被点名距今≥3课时 → 权重×1.5
  - 本课已被点名 → 权重×0.2（非零，保持"随机感"）
输出：按权重加权随机选中1人
```

#### 关注模式数据依赖

| 数据来源 | 判定条件 |
|---------|---------|
| 课堂参与记录 | 连续2课时参与次数为0 |
| 随堂测数据 | 近2次同知识点正确率下降≥20% |
| 积分数据 | 周积分排名在班级后20% |

### 6.2 策略切换入口

**入口设计：** 教师快捷栏"点名"按钮

- **短按**：使用当前策略立即发起点名（1步完成）
- **长按**：弹出策略选择面板

**策略选择面板：**
```
┌──────────────────────────────┐
│ 点名设置                      │
│                                │
│ 策略：                         │
│ ● 均衡模式（推荐）             │
│ ○ 关注模式                     │
│ ○ 纯随机                       │
│                                │
│ 动效：                         │
│ [常规点名]  [幸运星🌟]         │
│                                │
│         [确定]                 │
└──────────────────────────────┘
```

**设计约束：**
- 策略选择后持续生效，无需每次点名都选择
- 面板中当前策略用●标记，教师一眼可知当前状态
- "幸运星"动效选择也持续生效，直到教师再次切换

### 6.3 大屏动效 — 两种皮肤

**皮肤A：常规点名**

```
阶段1（1.5秒）：
┌─────────────────────────────────────────┐
│                                           │
│           姓名快速滚动中...               │
│           ┌─────────┐                    │
│           │ 张三     │ ← 快速滚动         │
│           └─────────┘                    │
│                                           │
└─────────────────────────────────────────┘

阶段2（停留）：
┌─────────────────────────────────────────┐
│                                           │
│              📢 有请                      │
│           ┌─────────────┐                │
│           │   李 小 红    │               │
│           └─────────────┘                │
│                                           │
│     ┌────┐  ┌────┐  ┌──────────┐       │
│     │ ✓对│  │ ✗错│  │ +分 [1▼] │       │
│     └────┘  └────┘  └──────────┘       │
└─────────────────────────────────────────┘
```

**皮肤B：幸运星**

```
阶段1（2秒）：
┌─────────────────────────────────────────┐
│         ✨  ✨  ✨  ✨  ✨               │
│                                           │
│           🌟 幸运星降临！ 🌟              │
│                                           │
│      [头像/姓名快速滚动......]            │
│                                           │
│         ✨  ✨  ✨  ✨  ✨               │
└─────────────────────────────────────────┘

阶段2（停留）：
┌─────────────────────────────────────────┐
│         ✨        ✨        ✨           │
│                                           │
│           🌟 恭喜 🌟                     │
│                                           │
│              李 小 红                     │
│           获得回答机会！⭐                 │
│                                           │
│     ┌────┐  ┌────┐  ┌──────────┐       │
│     │ ✓对│  │ ✗错│  │ +分 [2▼] │       │
│     └────┘  └────┘  └──────────┘       │
└─────────────────────────────────────────┘
```

**差异对比：**

| 维度 | 常规点名 | 幸运星 |
|------|---------|--------|
| 动效时长 | 1.5秒 | 2秒（稍长，增加仪式感） |
| 视觉风格 | 简洁、正式 | 星星粒子、金色调、活泼 |
| 文案 | "有请" | "恭喜！获得回答机会" |
| 默认加分 | +1 | +2（幸运星自带奖励加成） |
| 氛围 | 适合严肃提问 | 适合课堂活跃环节 |

### 6.4 点名后评价与加分

**流程：**
```
学生被选中 → 大屏展示姓名 → 学生口头回答 →
教师在大屏/小屏点击评价：
  - [✓ 对]：+1积分（常规）/ +2积分（幸运星）
  - [✗ 错]：+0积分（不扣分）
  - [+分]：额外加分（教师可选1-5分）
→ 积分动效 → 大屏回到课件/下一步
```

**教师小屏同步显示：**
```
┌─────────────────────────────────┐
│ 当前选中：李小红                  │
│                                   │
│ [✓对 +2]  [✗错]  [额外+分]      │
│                                   │
│ [跳过]                            │
└─────────────────────────────────┘
```

**"跳过"说明：** 学生不回答/教师不想评价时，点击跳过，不产生积分变动。

---

### 6.5 学生小屏体验

**被选中时：**
```
┌─────────────────────────────────┐
│                                   │
│      🌟 你被选中了！              │
│                                   │
│   请准备回答老师的问题            │
│                                   │
│          +2 积分已到账 ⭐         │
│                                   │
└─────────────────────────────────┘
```

**未被选中的学生：** 小屏无变化（不打扰）

## 七、异常与边界Case

| 场景 | 处理方式 |
|------|---------|
| 被选中的学生不在教室（请假/离线） | 大屏显示"该生不在线"→ 自动重新选中另一人（动效重新播放） |
| 均衡模式下所有学生本课都已被点过1次 | 降级为纯随机模式（本课已完成均衡覆盖） |
| 关注模式下无符合条件的学生（全班参与度均良好） | 降级为均衡模式，无特殊提示 |
| 教师连续点名同一学生（纯随机模式下） | 纯随机不干预，尊重概率结果 |
| 点名后教师忘记评价（既不点"对"也不点"错"） | 30秒后大屏自动收起点名面板，该次点名记录为"已点名/未评价" |
| 课堂只有5个学生在线（其他请假） | 正常从在线学生中选择，策略权重计算基于在线名单 |
| 策略数据依赖的课堂记录不完整（新学期开始） | 前2周统一使用纯随机模式积累数据，第3周起策略生效 |
| 教师在课中修改策略 | 立即生效，下一次点名使用新策略 |
